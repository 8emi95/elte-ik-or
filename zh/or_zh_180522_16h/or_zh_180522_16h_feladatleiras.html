Alapfeladat

Készíts dallamok írására egy szerver-kliens alkalmazást.

  • Készíts egy dallam.Hangjegy típust, ami egy adott ideig szóló hangjegzet reprezentál. Rendelkezzen két adattaggal: egy hang (String, egy szó), és egy hossz (int).
  • Készíts egy dallam.Dal típust, ami egy dalt repreyentál. Rendelkezzen két adattaggal: egy címmel (String), valamint hangjegyek listájával.
  • Készíts egy dallam.Szerver típust, ami egy futtatható Java osztály. A main függvény indítson egy szervert az első paranccsori paraméterként megadott porton, es várja egymás utáni kliensek csatlakozásat. A klienstől csatlakozás után előszőr egy sort várjon. ami a dal címét tartalmazza, majd ezután hangjegyek sorozatát. Egy hangjegy két szóból áll, az első szó a hangmagasságot, a második annak hosszát tartalmazza. A szerver nem küld válaszokat a kliensnek.
  • A szerver tárolja el a kapott dalokat egy adatszerkezetbe. Amennyiben egy dal korábban már létezett, azt írja felül.
  • Keszits egy dallam.Zeneszerzo nevű futtatható Java osztályt. A program kérjen be a felhasználótól a standard bemeneten egy címet, majd hangjegyek listáját a standard bemenet zárásáig, és azokat folyamatosan küldje el a paranccsori paraméterként megkapott szervernek (az első parancssori paraméter a host, a második a port)

Párhuzamosság

  • A szerverhez hallgatók is csatlakozhatnak, a második parancssori paraméterként megkapott porton. Ezt a funkcionalitást a dallam.Radio Runnable típus valósítsa meg.
    • Miután egy kliens kapcsolódott, el kell küldenie a szervernek melyik dalt akarja hallgatni.
    • Ezután bármikor elküldheti a szervernek a "figyel" üzenetet, amire a szerver az abban a pillanatban éppen lejátszott hang magasságával válaszol.
    • Amennyiben a dal már véget ért, vagy nem létezik, küldjön üres sort.
    • Párhuzamosan több hallgató is csatlakozhasson.
  • Készítsd el a dallam.Hallgato típust, ami egy futtatható Java osztály. A main függvény kapcsolódjon a paranccsori paraméterekkel (host, port) megadott szerverhez, majd a standard bemeneten megkapott címet (egy sor) küdje el a szervernek. Ezután minden harmadik másodpercben küldje el a figyel üzenetet a szervernek, és jelenítse meg a kapott választ.
  • A megoldás legyen szálbiztos: egy rossz ütemezés ne okozhassa a program leállását vagy hibás működését!

RMI

  • A szerver vezessen egy statisztikát arról, melyik hangot hányszor használták, a dallam.IStat távoli interface segítségével.
  • Az interface tartalmazzon egy olyan metódust, ami visszaadja, hogy a paraméterként megkapott magasságú hangot hányszor tartamazzák dalok. A metódus paramétereként kapja meg azt is, hogy az aktuális dal hányszor használta fel ezt a hangot.
  • A dallam.Stat osztály implementálja ezt az interface-t. A Szerver típus mindig frissítse ezt az információt, miután egy hangot hozzáadott.
  • A szerver exportálja ezt az RMI szolgáltatást egy névjegyzékbe. A névjegyzéket a 31138 porton indítsd el.
  • Készíts egy dallam.HangInfo futtatható Java osztályt, ami megjeleníti a paranncsori paraméterként megkapott hangjegyhez tartozó darabszámot.

Adatbázis

  • Tárold el adatbázisban, melyik dalt hányszor játszották le egy két oszlopot (cím, hányszor) tartalmazó táblában.
  • Az adatbázis műveleteket a dallam.Model típusban található metódusok valósítsák meg.
  • Amennyiben a tábla nem létezik, azt a modell típus automatikusan hozza létre.
  • A Radio típus frissítse ezt az információt, amikor egy dalt elkezd lejátszani.
  • Készíts egy dallam.Toplista futtatható Java osztályt, ami megjeleníti a három legtöbbet lejátszott dalt, és hogy hányszor játszották le őket.
